Agent Reach Skill：让 Agent 看见真实世界

这节课不讲后台实现细节，而是讲一个普通用户如何把 Agent Reach 接到自己的 Agent 上：给安装链接、打开执行权限、装浏览器桥接工具、登录 X / 小红书，再让 Agent 读取真实平台内容。

开场 Agent 最大的问题不是不会写

Agent 最大的问题，是看不见现场

你让 AI 分析 X 上的讨论、小红书上的评价，或者某个账号最近的内容，如果它没有真实读取平台，只能靠常识猜。Agent Reach 要解决的第一件事，就是让 Agent 能进入外部平台，把现场信息拿回来。

Without Reach

复制链接给 AI

AI 可能打不开，或者只能根据标题猜。你得到的是“像分析”的文字。

VS
With Reach

让 Agent 读取现场

Agent 通过正确通道读取平台内容，再把结果交给 Codex 做判断。

第一性原理 用户到底要装什么

不是装一个工具，而是打通三层东西

从用户视角看，Agent Reach 的安装不是为了背命令，而是给 Agent 装上“读取外部世界”的三层脚手架。

01

Agent 端

把 Agent Reach 安装链接发给 Codex / Claude Code / OpenClaw / Cursor，让 Agent 学会这套 Skill。

02

浏览器端

对 X、小红书这类需要登录态的平台，需要 Chrome + OpenCLI / OpenClaw 这类浏览器桥接能力。

03

账号端

你要先在浏览器里登录平台。Agent 不是绕过登录，而是读取你已经能看到的公开内容。

第一性原理：Agent 只能分析它能读取到的现场。

前台流程 用户真正要做的几步

先安装，再连接浏览器，最后跑平台任务

这部分才是课堂里应该讲的“操作路径”。它不是 Agent 后台的路由表，而是你作为使用者怎样一步一步把能力接起来。

install instruction

可以直接丢给 Codex 的第一句话

Install Agent Reach: https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md

如果你用的是 OpenClaw，要先确认它允许执行 shell 命令；否则 Agent 没法安装依赖和跑 doctor。

示例 01 X / Twitter 怎么跑

X 的重点：搜索、时间线和单条推文

X 的课程案例适合讲“公开讨论调研”：看某个工具、话题、人物或趋势在海外技术圈怎么被讨论。

准备

先登录 X

在 Chrome 里登录 X。若走 twitter-cli，可能需要 cookie；若走 OpenCLI，则复用浏览器登录态。

任务

让 Agent 读讨论

例如：搜索 “Agent Reach”，读取 10 条相关讨论，保留链接、作者、发布时间和观点。

输出

不要只总结

输出“大家在夸什么、担心什么、哪些观点值得引用、哪些只是噪声”。

example task / X

“请用 Agent Reach 读取 X 上关于 Agent Reach 的近期讨论。先说明你使用的后端和读取范围，再给我 5 条代表观点和链接，最后总结正面评价、负面顾虑和可引用金句。”

示例 02 小红书怎么跑

小红书的重点：Chrome 登录态 + OpenCLI

小红书不是“直接拿 note_id 就读”。正确方式通常是先搜索或打开 feed，拿到完整结果和 token，再读笔记、评论或用户主页公开内容。

Step 1

用小号登录 Chrome

小红书有风控，课堂演示建议用小号。不要用主账号做高频抓取。

Step 2

装浏览器桥接

通过 OpenCLI / OpenClaw 这类工具，让 Agent 能复用你浏览器里已经登录的小红书页面。

Step 3

先搜索，再读取

先搜索关键词或账号，再基于搜索结果读取笔记详情、评论和用户主页公开内容。

example task / 小红书

“请用 Agent Reach 的小红书通道搜索【AI 学习 / Agent Reach / Codex】相关内容。先确认 active_backend，再读取 10 条代表笔记，输出标题、账号、互动数据、主要观点和可迁移选题。”

能看见真实世界，也意味着要知道哪里不能乱碰。

只读优先课程演示只做读取、搜索、总结，不做发帖、评论、点赞。
→
小号演示涉及小红书等风控平台，用小号登录，降低账号风险。
→
证据留痕保留读取范围、来源链接、时间和限制，方便复盘。
课堂练习 从 X 到小红书跑一次

练习：同一个问题，两个平台怎么看

选一个你关心的工具或话题，比如 Agent Reach、Codex、某个 AI 产品。让 Agent 分别读取 X 和小红书，比较两个平台的讨论差异。

X 侧问题

技术圈怎么评价？关注点是安装、能力、可靠性、替代方案，还是安全边界？

小红书侧问题

普通用户怎么理解？他们关心的是教程、案例、好不好用，还是账号风险？

输入同一个主题 + 两个平台 + 读取范围。
过程先确认后端，再读取内容，再保留来源。
输出平台差异、代表观点、可引用素材、下一步选题。
课后 作业

课后作业：完成一次前台接入

这次作业不是写一段漂亮指令，而是把 Agent Reach 真的接起来，并用 X 或小红书跑一次只读任务。

安装把 Agent Reach 安装链接发给 Codex / 其他 Agent，让它完成安装和 doctor 检查。
浏览器在 Chrome 中准备 OpenCLI / OpenClaw 这类浏览器桥接能力，并登录演示平台。
X 任务读取一个话题的近期讨论，输出代表观点和证据链接。
小红书任务用小号登录后，读取一个关键词下的代表笔记，输出可迁移选题。
复盘记录哪里卡住：权限、登录态、风控、后端不可用，还是任务描述不清楚。
